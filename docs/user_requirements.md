# User Requirements

## Admin
1. The Admin shall be able to log in to a secure dashboard.
2. The Admin shall be able to view a list of pending events.
3. The Admin shall be able to approve or reject events submitted by organizers.
4. The Admin shall be able to view system-wide statistics (e.g., total users, total events).
5. The Admin shall be able to manage users (view list, update roles).
6. The Admin shall be able to delete any event if necessary.

## Organizer
1. The Organizer shall be able to log in to a secure dashboard.
2. The Organizer shall be able to create new events with details such as title, description, date, time, location, and capacity.
3. The Organizer shall be able to edit or delete their own events.
4. The Organizer shall be able to view the status of their submitted events (pending, approved, rejected).
5. The Organizer shall be able to track RSVPs and view attendee lists for their events.
6. The Organizer shall be able to check in guests using a digital check-in system.
7. The Organizer shall be able to view feedback submitted by guests.
8. The Organizer shall be able to receive notifications regarding event approval and new feedback.
9. The Organizer shall be able to customize their profile and preferences (branding, accent colors).
10. The Organizer shall be able to view earnings and payout analytics for paid events.

## Student (Guest)
1. The Student shall be able to register and log in to the system.
2. The Student shall be able to browse and search for approved public events.
3. The Student shall be able to view detailed information about an event, including location maps.
4. The Student shall be able to RSVP for free events.
5. The Student shall be able to purchase tickets for paid events using a secure payment gateway.
6. The Student shall be able to view their tickets and RSVP status.
7. The Student shall be able to use a QR code or digital ID for event check-in.
8. The Student shall be able to submit feedback (rating and comment) after attending an event.
9. The Student shall be able to manage their profile, including profile picture and interests.
10. The Student shall be able to receive notifications for RSVP confirmations and reminders.
